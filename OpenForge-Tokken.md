## When Do You Need to Import a Token?

If a project distributes **custom tokens** (ERC-20 or similar), they may **not appear automatically** in your wallet.
In that case, you must **manually import the token** using its contract address.

Project owners will share:

* Token contract address
* Network (usually Sepolia)
* Token symbol and decimals (if needed)

---

## Requirements

* Browser wallet: **MetaMask**
* Correct network selected (usually **Sepolia**)
* Token contract address from the project owner

## Important Notes About Tokens on OpenForge

Please read carefully:

* Tokens are **issued by individual projects**, not OpenForge
* Token value is **not guaranteed**
* Tokens may have **no liquidity**
* Tokens may be **reset, changed, or deprecated**
* OpenForge does **not verify or audit project tokens**

Always clarify terms **before contributing**.

## Common Issues & Fixes

**Token not showing after import?**

* Double-check the network
* Confirm the contract address
* Make sure tokens were actually sent

**Wrong token imported?**

* Remove it from MetaMask
* Re-import using the correct address
